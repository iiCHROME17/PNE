PNE API v1.0

PNE API — Unity Integration Guide
Connect Godot's rival to the Psychological Narrative Engine

Companion files: docs/unity/PNETypes.cs  ·  docs/unity/PNEClient.cs  ·  docs/unity/PNEDialogueUI.cs

Overview

The Psychological Narrative Engine (PNE) runs as a local HTTP + WebSocket server. Unity connects to it to drive BDI-powered NPC conversations:

  • Session management via REST — create sessions, list NPCs and scenarios.
  • Turn gameplay via WebSocket — send player choices, receive streaming NPC dialogue and full BDI state.

A single conversation turn flows like this:

Unity                             PNE Server
  │── POST /sessions ──────────►  load NPCs + scenario, create session
  │◄── { session_id, choices } ─  opening choice list returned
  │
  │══ WS /sessions/{id}/play ══►  WebSocket connection opened
  │── { "choice_index": 2 } ────►  player picks choice 2
  │◄── { "type": "player_choice" } echo player's line immediately
  │◄── { "type": "token" } ─────  Ollama streams word by word (2–10 s)
  │◄── { "type": "token" } ─────
  │◄── { "type": "turn_result" }  BDI + dice roll data
  │◄── { "type": "choices" } ───  next set of choices
  │        … repeat per turn …
  │◄── { "type": "terminal" } ──  conversation ends; NPC JSONs auto-saved

Starting the Server

On the machine running the PNE (can be the same machine as Unity):

bash 
cd path/to/cs3ip/Models
pip install fastapi "uvicorn[standard]"

uvicorn api.main:app --host 0.0.0.0 --port 8000

Verify it's running by opening http://localhost:8000/docs in a browser — you'll see the interactive Swagger UI.

Tip: Add --reload during development so the server restarts automatically when Python files change.

Unity Project Setup

1. Install required packages

Open Window → Package Manager.

Input System

  1. Click + → Add package from registry
  2. Search for Input System and click Install
  3. When prompted to switch active Input Handling, click Yes — this sets Player Settings to Input System Package (New)

NativeWebSocket

  1. Click + → Add package from git URL
  2. Paste: https://github.com/endel/NativeWebSocket.git#upm
  3. Click Add

Newtonsoft Json (JSON.NET)

  1. Click + → Add package from git URL
  2. Paste: com.unity.nuget.newtonsoft-json
  3. Click Add

All three packages are required. Input System provides keyboard handling in PNEDialogueUI (E / Space to skip or advance). NativeWebSocket handles the WebSocket streaming (including WebGL). Newtonsoft Json parses the dynamic API messages reliably — Unity's built-in JsonUtility cannot handle the flexible shapes the server sends.

2. Copy the C# files into your project

Copy all three files from Models/docs/unity/ into your Unity project's Assets/ folder (any subfolder is fine):

text 
Assets/
  PNE/
    PNETypes.cs        ← Data classes matching all API JSON shapes
    PNEClient.cs       ← MonoBehaviour — HTTP + WebSocket client
    PNEDialogueUI.cs   ← Example UI controller

3. Build settings

WebGL: no extra steps — NativeWebSocket handles platform differences automatically.
Standalone (Windows / Mac / Linux): everything works out of the box.

PNEClient Reference

Events

Subscribe in Awake or Start on any script:

C# 
var pne = GetComponent<PNEClient>();

pne.OnSessionReady   += data        => Debug.Log(data.Scenario.Opening);
pne.OnPlayerChoice   += msg         => AppendConvoLine("You", msg.Text);
pne.OnToken          += (npc, tok)  => dialogueText.text += tok;
pne.OnTurnResult     += msg         => Debug.Log(msg.Thought.SubjectiveBelief);
pne.OnChoicesUpdated += msg         => RebuildChoiceButtons(msg.Choices);
pne.OnTerminal       += msg         => ShowOutcome(msg.TerminalId, msg.Result);
pne.OnError          += err         => Debug.LogError(err);

Methods

C# 
pne.StartSession();          // Creates session and opens WebSocket
pne.SendChoice(int index);   // 1-based index from ChoiceItem.Index
pne.EndSession();            // Closes WS and deletes session from server

Properties

C# 
pne.SessionId    // string — UUID of the active session
pne.IsConnected  // bool   — WebSocket is open and ready
pne.IsComplete   // bool   — terminal outcome has been reached

WebSocket Message Reference

Every message from the server has a "type" field. PNEClient deserialises these automatically and fires the matching event.

"player_choice" → OnPlayerChoice(PlayerChoiceMessage)

Sent immediately when the player's choice is received — before any Ollama tokens arrive. Use this to add the player's line to a conversation log so the display updates right away.

JSON 
{
  "type": "player_choice",
  "choice_id": "open_empathy",
  "text": "I've seen what this regime does to families.",
  "language_art": "empathetic"
}

UI pattern: append this as the player's line in a chat-style log before the NPC starts typing. Disable choice buttons at this point — re-enable them in OnChoicesUpdated.

"token" → OnToken(npcName, token)

One chunk of Ollama's streaming output.

JSON 
{ "type": "token", "npc": "Troy", "token": "I " }
{ "type": "token", "npc": "Troy", "token": "don't " }
{ "type": "token", "npc": "Troy", "token": "trust you." }

UI pattern (PNEDialogueUI): tokens are buffered silently in _npcBuffer. When OnTurnResult fires, the complete response is typewriter-animated via maxVisibleCharacters. This avoids partial-word flicker and lets the player skip or wait.

Custom pattern (live streaming): if you prefer tokens displayed as they arrive:

pne.OnToken += (npc, token) => dialogueText.text += token;

"turn_result" → OnTurnResult(TurnResultMessage)

Sent after all Ollama tokens for a turn have been streamed. Contains the full BDI state plus dice roll data.

JSON 
{
  "type": "turn_result",
  "npc": "Troy",
  "npc_id": "troy",
  "thought": {
    "subjective_belief": "This person is testing me.",
    "emotional_valence": -0.45
  },
  "desire": { "desire_type": "information-seeking", "intensity": 0.72 },
  "intention": { "intention_type": "Evaluate Sincerity", "confrontation_level": 0.6 },
  "judgement": 42,
  "npc_response": "I don't trust you.",
  "dice": {
    "player_die": 3,
    "npc_die": 5,
    "success": false,
    "skill": "empathy",
    "success_pct": 45,
    "risk_multiplier": 1.11,
    "judgement_delta": -6
  },
  "entered_recovery": true
}

UI pattern: update a judgement bar with judgement (0–100), or use emotional_valence to tint the dialogue panel (see Common UI Patterns).

Arcade / Disco Elysium mode — dice fields:

  • dice.player_die / dice.npc_die — raw 1–6 rolls to display on screen.
  • dice.success — whether the player's roll beat the NPC's.
  • dice.skill — which player skill was tested (e.g. "empathy").
  • dice.success_pct — the pre-roll probability shown on the choice button.
  • dice.judgement_delta — how many judgement points were gained or lost this turn (show as ±N).
  • dice.risk_multiplier — scales delta when odds were against the player (value > 1.0 = high-risk roll).
  • entered_recovery — true when this failure triggers a recovery sub-turn. Show a recovery indicator on the next choices panel.

When use_ollama = false, dice may be null for scripted nodes.

"choices" → OnChoicesUpdated(ChoicesMessage)

The next set of available player choices. Rebuild your choice buttons when this fires.

JSON 
{
  "type": "choices",
  "node_id": "challenge",
  "in_recovery": false,
  "choices": [
    { "index": 1, "choice_id": "press_harder", "text": "I have proof.",
      "language_art": "challenge",  "success_pct": 58 },
    { "index": 2, "choice_id": "back_off",     "text": "Fair enough.",
      "language_art": "diplomatic", "success_pct": 71 }
  ]
}

in_recovery — when true, the player is in a recovery sub-turn after a failed dice check. Show a visual indicator (e.g. an amber border around the choices panel) so the player understands they are recovering, not choosing freely.

"terminal" → OnTerminal(TerminalMessage)

Conversation over. Show the outcome, stream the final NPC line, and disable choices.

JSON 
{
  "type": "terminal",
  "npc": "Troy",
  "terminal_id": "SUCCEED",
  "result": "Troy steps aside and lets you through.",
  "final_dialogue": "...don't make me regret this."
}

Possible terminal_id values:

SUCCEED FAIL NEGOTIATE DELAY ESCALATE

"error" → OnError(message)

Something went wrong server-side. Log it and optionally show the player a retry option. Errors do not close the WebSocket — the session can continue if the error was non-fatal.

success_pct on choices

Each ChoiceItem carries a success_pct (0–100) so you can show the player their odds. This accounts for their relevant skill vs the NPC's resistance, plus the current player_relation which drifts up as the NPC warms to the player.

Common UI Patterns

Typewriter with a scroll view

PNEDialogueUI uses TMP_Text.maxVisibleCharacters for the typewriter — this respects all TMP rich text tags (<b>, <color>, <size>) without string-slicing issues. To pair with a ScrollRect, force the canvas after each tick:

C# 
// In a custom TypewriterRoutine override:
dialogueText.maxVisibleCharacters = i;
Canvas.ForceUpdateCanvases();
scrollRect.verticalNormalizedPosition = 0f;
yield return new WaitForSeconds(delay);

Set ScrollRect → Content to the DialogueText panel with a Content Size Fitter (Preferred Size, vertical).

Emotion-based panel tinting

Use emotional_valence from TurnResultMessage to tint the dialogue panel:

C# 
private void HandleTurnResult(TurnResultMessage msg)
{
    float v = msg.Thought?.EmotionalValence ?? 0f;
    // +1.0 = warm/positive, -1.0 = cold/hostile
    dialoguePanel.color = Color.Lerp(hostileColor, friendlyColor, (v + 1f) / 2f);
}

Judgement bar

Drive a Slider or Image.fillAmount from judgement (0–100), and show the per-turn delta:

C# 
private void HandleTurnResult(TurnResultMessage msg)
{
    judgementBar.value = msg.Judgement / 100f;

    if (msg.Dice != null)
    {
        int delta = msg.Dice.JudgementDelta;
        deltaLabel.text = delta >= 0 ? $"+{delta}" : delta.ToString();
        deltaLabel.color = delta >= 0 ? Color.green : Color.red;
    }
}

Conversation log (player + NPC lines)

Build a chat-style history by appending to a scrollable log on both OnPlayerChoice and OnTurnResult:

C# 
void Start()
{
    pne.OnPlayerChoice += msg => AppendLog("You", msg.Text);
    pne.OnTurnResult   += msg => AppendLog(msg.Npc, msg.NpcResponse);
}

void AppendLog(string speaker, string line)
{
    var entry = Instantiate(logEntryPrefab, logContainer);
    entry.GetComponentInChildren<TMP_Text>().text = $"<b>{speaker}:</b> {line}";
    Canvas.ForceUpdateCanvases();
    logScrollRect.verticalNormalizedPosition = 0f;
}

Arcade mode — dice roll display

Show the raw dice values and outcome after each turn (similar to Disco Elysium). The dice field in TurnResultMessage has everything you need:

C# 
private void HandleTurnResult(TurnResultMessage msg)
{
    if (msg.Dice == null) return;

    // Animate dice faces to the rolled values
    playerDieDisplay.SetFace(msg.Dice.PlayerDie ?? 0);
    npcDieDisplay.SetFace(msg.Dice.NpcDie ?? 0);

    // Show success / failure label
    rollResultLabel.text   = msg.Dice.Success ? "SUCCESS" : "FAILURE";
    rollResultLabel.color  = msg.Dice.Success ? Color.green : Color.red;

    // Show pre-roll odds and risk
    oddsLabel.text = $"{msg.Dice.SuccessPct}% — {msg.Dice.Skill}";
    if (msg.Dice.RiskMultiplier > 1f)
        riskLabel.text = $"HIGH RISK ×{msg.Dice.RiskMultiplier:F2}";

    // Show recovery indicator if the failed roll triggers a recovery sub-turn
    recoveryIndicator.SetActive(msg.EnteredRecovery);
}

Tips & Gotchas

NativeWebSocket callbacks are already on the main thread

NativeWebSocket uses DispatchMessageQueue() in Update() to marshal callbacks onto the main thread. You do not need MainThreadDispatcher or lock statements around the events fired by PNEClient.

Ollama takes 2–10 seconds per turn

Disable choice buttons as soon as SendChoice() is called — this is already done in PNEDialogueUI. Re-enable them in OnChoicesUpdated. Never call SendChoice() twice in the same turn.

Server must be reachable by IP

On mobile or other devices, use the host machine's LAN IP (192.168.x.x) instead of localhost in the Api Base Url field.

Session cleanup

PNEClient.OnDestroy() calls EndSession() automatically, so sessions are cleaned up when the scene changes. For explicit control, call client.EndSession() before loading a new scene.

Multiple NPCs

When multiple NPCs are active, OnToken and OnTurnResult fire once per NPC per turn. Use msg.Npc or msg.NpcId to route to the correct UI element (e.g. a separate text box per NPC portrait).

Offline / no Ollama mode

Set Use Ollama = false in the Inspector. The engine falls back to the scenario's scripted min_response / max_response text — fast and deterministic. Useful for testing UI without running Ollama locally.

Troubleshooting

SymptomFix
Session creation failed (HTTP 0) Server not running, or wrong apiBaseUrl
Session creation failed (HTTP 400) NPC / scenario path wrong — check they are relative to Models/
WebSocket never connects Firewall blocking port 8000; confirm server was started with --host 0.0.0.0
No tokens arrive, but turn_result fires use_ollama = false set, or Ollama not running at localhost:11434
NativeWebSocket namespace not found Package not installed — re-add the git URL in Package Manager
Newtonsoft.Json not found Add com.unity.nuget.newtonsoft-json in Package Manager
InvalidOperationException: You are trying to read Input using the UnityEngine.Input class Go to Edit → Project Settings → Player → Active Input Handling and set it to Input System Package (New) or Both. Install the Input System package if missing.

Choices panel blank after a turn Check choicesContainer and choiceButtonPrefab are assigned in the Inspector
Text stops streaming mid-sentence Ollama model may have timed out (60 s); check the server console for errors
